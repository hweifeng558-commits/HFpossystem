園藝 POS 系統 — PWA 安裝說明
================================

這個資料夾裡有 4 樣東西：
- index.html      App 本體
- manifest.json   App 的名稱、圖示、顯示模式設定
- sw.js           離線快取用的 Service Worker
- icons/          App 圖示 (192px / 512px)

這四個檔案的相對位置不能變，整包上傳、放在同一層資料夾即可。

── 第一步：把資料夾放到網路上（擇一即可，都免費）──

方法 A：Netlify（最快，拖拉就好）
1. 瀏覽器打開 https://app.netlify.com/drop
2. 把整個資料夾拖進網頁裡
3. 幾秒後會拿到一個網址，例如 https://xxxx.netlify.app
4. 完成，之後每次都用這個網址開啟

方法 B：GitHub Pages
1. 在 GitHub 建一個新的 repository
2. 把這四個檔案（含 icons 資料夾）上傳上去
3. 到 repository 的 Settings → Pages，把 Source 設成 main branch
4. 幾分鐘後會拿到網址，例如 https://你的帳號.github.io/repo名稱/

（PWA 一定要放在 https 網址上才能真正「安裝」，用本機檔案打開沒辦法。）

── 第二步：在收銀用的平板/手機/電腦上安裝 ──

iPhone / iPad (Safari)：
打開網址 → 點下方分享圖示 → 「加入主畫面」

Android (Chrome)：
打開網址 → 右上角選單 → 「安裝應用程式」或「加到主畫面」

電腦 (Chrome / Edge)：
打開網址 → 網址列右側會出現「安裝」圖示 → 點下去

安裝後桌面/主畫面會出現一個綠色 H 的圖示，點開就是全螢幕、
沒有網址列的畫面，之後就算網路不穩或暫時斷線，也能照常結帳
（第一次要在有網路時開啟過一次，才會把檔案存進裝置）。

── 注意事項 ──

1. 資料儲存在「這台裝置的這個瀏覽器」裡（localStorage）。
   如果有多台收銀機，彼此的商品資料、銷售紀錄不會互相同步，
   是各自獨立的。如果未來需要多機共用同一份資料，需要另外
   接資料庫，是更大的工程，有需要再跟我說。

2. 如果之後要更新程式內容（例如改折扣邏輯），改好 index.html
   後重新上傳覆蓋即可，使用者端下次連網開啟會自動抓到新版本。

3. 清除瀏覽器資料/App 資料會把商品和交易紀錄一起清掉，
   建議定期用「報表統計」頁面的「匯出報表」備份成 Excel。
